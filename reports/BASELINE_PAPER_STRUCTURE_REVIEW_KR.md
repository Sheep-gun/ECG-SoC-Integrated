# 기준 논문 서술 구조 분석 및 통합 보고서 적용 기록

## 1. 분석 대상과 범위

- 논문: H. Park 외, “A CMOS Hybrid System for Non-Invasive Hemoglobin and Oxygen Saturation Monitoring with Super Wavelength Infrared Light Emitting Diodes,” *Micromachines*, vol. 16, 1086, 2025.
- 원본: `micromachines-16-01086.pdf`, 총 19쪽.
- 검토 범위: 제목, 초록, 서론, 관련 연구, 회로 설명, 디지털 회로, 시뮬레이션 결과, 논의, 결론과 참고문헌을 포함한 전 페이지.
- 검토 방법: 페이지별 본문 추출과 전 페이지 이미지 렌더링을 함께 사용하여 문장 내용뿐 아니라 표·그림의 배치와 절 전환도 확인하였다.

이 문서는 기준 논문의 기술 내용을 본 프로젝트의 근거로 사용하는 문서가 아니다. 논문의 서술 순서, 문장 역할과 단락 짜임을 분석하여 통합 기술보고서의 가독성과 완결성을 높이기 위한 편집 기준이다.

## 2. 논문의 전체 서술 구조

기준 논문은 다음 순서로 전개된다.

1. **초록:** 측정 대상과 기존 장치의 한계를 먼저 말한 뒤, 제안 시스템의 센서·아날로그·디지털 구성을 한 문단에 요약하고 핵심 정확도로 마무리한다.
2. **서론:** 생체 신호의 필요성을 설명하고, PPG의 AC/DC 성분과 파장별 흡수 특성을 이용해 왜 다중 파장이 필요한지 단계적으로 좁힌다.
3. **관련 연구:** 2파장 방식의 한계, 3파장 결정계의 잡음 민감도, 5파장 과결정계의 장점을 식과 선행 결과로 연결한 뒤 제안 구조의 위치를 밝힌다.
4. **회로 설명:** 시스템 수준 블록도를 먼저 제시하고 정량 목표 표를 둔 다음, LED driver, receiver, digital processing을 신호 흐름 순서로 설명한다.
5. **블록별 설명:** 각 절은 블록의 목적, 회로 또는 수식, 구현값, 출력이 다음 블록에 어떻게 쓰이는지를 차례로 제시한다.
6. **결과:** 아날로그 post-layout 결과와 디지털 구현 결과를 표로 모으고 무엇과 비교했는지를 함께 쓴다.
7. **논의:** 알고리즘 선택, 아날로그 회로, Python/MATLAB 교차검증, Verilog 검증을 검증 방법별로 분리한다.
8. **결론:** 제안 시스템의 구성, 핵심 회로 수치, 디지털 결과, 비교표, 전력 한계와 wearable 적용 가능성을 순서대로 요약한다.

핵심은 절의 수가 아니라 질문의 순서다. 독자가 먼저 “왜 필요한가”를 이해한 뒤 “무엇을 제안했는가”, “각 블록이 어떻게 동작하는가”, “어떻게 검증했는가”, “수치가 무엇을 뜻하는가”를 따라가게 만든다.

## 3. 문장과 단락의 공통 형태

기준 논문의 기술 단락은 대체로 다음 네 문장 역할을 반복한다.

1. **목적 문장:** 해당 블록이 해결하는 문제를 첫 문장에 제시한다.
2. **동작 문장:** 입력이 어떤 계산이나 회로를 거치는지 설명한다.
3. **구현 문장:** 주파수, 비트 폭, 차수, 전류, 면적 등 실제 설계값을 제시한다.
4. **결과 문장:** 그림·표·시뮬레이션이 무엇을 확인했는지 적는다.

한 단락에 새로운 개념을 여러 개 쌓기보다 같은 질문에 답하는 문장끼리 묶는다. 수식이나 표 앞에서는 왜 그 표현이 필요한지 설명하고, 뒤에서는 수식이나 표가 보여주는 결론을 한 문장으로 닫는다. 그림은 장식이 아니라 시스템 경계, 회로 구성 또는 검증 결과를 설명하는 위치에 배치한다.

## 4. 통합 보고서에 적용한 구조

기준 논문의 흐름을 본 프로젝트에 맞춰 다음처럼 대응시켰다.

| 기준 논문 역할 | 통합 보고서 대응 |
|---|---|
| 서론의 측정 필요성 | 장시간 ECG에서 세 시간척도가 필요한 이유 |
| 관련 연구의 방식 비교 | 전체 저장·짧은 구간 회로·학습형 SNN의 한계와 설계 요구 |
| 시스템 블록도와 목표 표 | MATLAB→LTspice→XMODEL→signed stream→RTL/IP→FPGA 흐름과 인터페이스 |
| 회로 설명 | 표본값→사건→박동→리듬·파형→Snapshot→Final Membrane |
| post-layout/algorithm verification | MATLAB 공칭 AFE, LTspice schematic, XMODEL, RTL/XSim, Vivado와 FPGA 기능 등가성 |
| 결과 표 | 분류, 혼합신호 인계와 하드웨어 구현 결과의 분리 |
| discussion | 차별성, 데이터 한계, 물리·임상·physical board benchmark 공백 |

다음 편집 원칙도 반영하였다.

- 초록을 배경·제안·구조·검증·결과·한계 순서로 다시 구성하였다.
- 서론에서 장시간 ECG의 필요성과 회로 설계 문제를 분리하였다.
- 제2장을 관련 접근의 한계에서 설계 요구를 도출하는 장으로 바꾸었다.
- 제3장의 세부 회로를 3단계 번호로 나누어 목적별 탐색이 가능하게 하였다.
- 제4장과 제5장 앞에 장의 질문과 결과 분류 원칙을 명시하였다.
- 수치가 나올 때마다 평가 대상과 증명 범위를 함께 적는 기존 claim 경계를 유지하였다.
- 기준 논문에 없는 프로젝트 수치나 물리 검증을 새로 만든 것처럼 표현하지 않았다.

## 5. 의도적으로 그대로 따르지 않은 부분

기준 논문은 동일한 배경 설명이나 성능 해석을 여러 절에서 반복하고 일부 결과를 넓게 해석한다. 통합 보고서는 대회 심사 문서라는 목적에 맞춰 다음 사항은 따르지 않았다.

- 같은 수치의 반복을 줄이고 상세 경로는 부록과 evidence map으로 보냈다.
- 모델 기반 AFE 검증을 physical analog 또는 post-layout 결과로 표현하지 않았다.
- FPGA 기능 등가성 36/36을 분류 정확도 100%로 해석하지 않았다.
- 실보드 counter timing, Vivado 추정전력과 physical board 전력 미측정·임상·ASIC 공백을 결론에서도 숨기지 않았다.
- 영어 전문용어는 처음 정의할 때만 병기하고 이후에는 가능한 한 한글 설명을 먼저 사용하였다.

따라서 이번 개정의 목표는 기준 논문의 외형을 모방하는 것이 아니라, 독자가 기술적 필요성에서 회로 구조와 검증 결과까지 끊김 없이 따라가게 만드는 데 있다.
