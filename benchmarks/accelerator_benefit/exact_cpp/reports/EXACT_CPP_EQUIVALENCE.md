# Exact C++ equivalence

## Reference hierarchy

Locked full-top XSim at `sample_gap_cycles=2` is the final authority. Existing committed final outputs establish the 36-case targets. Verification-only testbenches expose internal state without changing RTL, parameters, accepted-sample timing, or outputs. The handwritten C++ model is never linked to or generated by a simulator.

The historical Python cycle model was used to parse constants and understand modules, but it was not treated as final truth where it disagreed with XSim. Full-top tracing identified current-versus-delayed sample alignment, rhythm-only PNN evaluation, boundary cadence, pre-commit RBBB gating, and C24 readout capture behavior. Those are scheduling corrections, not classifier tuning.

## Golden Snapshot trace

The golden was generated from the locked full top with verification-only instrumentation and contains exactly 36×30 rows. SHA-256:

`7fbdaf2a4a182a3c2757e7c7f923ff857b79d96756b509f75c588db515717a13`

The normalized fields are case/class, boundary index, accepted count, Snapshot prediction, four C24 membranes, available feature counters/evidence summaries, four post-accumulation final membranes, and structural-gate bits. No unavailable state was invented.

## Accepted-sample hash schema

FNV-1a with fixed little-endian serialization covers persistent event state, QRS membrane/refractory, PNN token/age/interval/predictor/evaluation state, RDM prior RR, ectopic reference, DSCR filter/up/down state, RAM peak/post-hold, QRS-MAF window/reference, and RBBB activity/age/count. Physical class-score/readout pipeline registers and aggregate trace counters are excluded because their schedule is deliberately compressed; scores/counters are instead compared exhaustively at boundaries.

Representatives are NSR `16272_chunk03`, CHF `chf07_chunk09`, cadence-sensitive ARR `118_chunk00`, and boundary-sensitive AFF `06995_chunk12`, first 60,000 accepted samples each. Each expected XSim hash-stream digest equals the C++ digest and no per-sample mismatch occurs.

## Final exactness

Every case has exactly 1,800,000 accepted samples, 30 Snapshot commits, and one final decision. Every expected/actual final field is integer-identical. Debug and release output hashes also match for every case. Therefore the achieved term is **exact C++ baseline**, not merely output-compatible or boundary-compatible.

## Remaining differences and limitations

There is no unresolved difference known to affect feature state, Snapshot output, accumulated final state, final prediction, or final membrane values. The C++ object graph does not reproduce pure physical RTL staging, simulator delta cycles, or clock toggling. Accepted-sample hash coverage is representative rather than all 64.8 million case samples; all 64.8 million samples nonetheless contribute to the all-case Snapshot and final proof.

The available MinGW lacks sanitizer runtime libraries; sanitizer configuration fails at link time and is documented rather than silently skipped. No runtime, throughput, latency, power, energy, or speedup measurement was performed.
