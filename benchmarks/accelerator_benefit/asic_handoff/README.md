# ASIC PPA Handoff (Not Yet Characterized)

This directory packages the locked accelerator RTL for a target-library ASIC
synthesis and post-layout power run. It does **not** contain fabricated 55/65/28
nm results: no licensed standard-cell Liberty/LEF files, SRAM macros, RC corner,
or sign-off power tool are installed in this workspace.

## Locked scope

- top: `snn_ecg_30min_final_top`
- functional clock target: 100 MHz (10 ns)
- RTL file order: `rtl_files.f`
- synthesis constraint: `snn_ecg_100mhz.sdc`
- activity inputs: use the committed real-ECG SAIF files generated by
  `../power/capture_ecg_activity.py`
- legacy firmware/report label `AFF` is preserved internally; report-facing
  medical label is AF.

## Required foundry inputs

For every claimed process node and PVT corner, provide:

1. standard-cell Liberty files with internal power and leakage tables;
2. technology LEF plus standard-cell LEF and any SRAM macro views;
3. RC technology/corner files;
4. antenna/filler/tap/endcap and power-grid rules;
5. operating voltage, temperature, process corner, and clock uncertainty;
6. retention, isolation, level-shifter and power-switch cells if power gating is
   claimed;
7. a sign-off-capable power engine and the real-ECG SAIF mapped to the post-route
   netlist.

## Reproducible stage gates

| Gate | Required evidence |
|---|---|
| Synthesis | mapped netlist, cell area, worst slack, Liberty/SDC hashes |
| Place/route | routed DEF/GDS, DRC/LVS status, extracted parasitics |
| Post-layout timing | setup/hold reports for declared PVT corners |
| Post-layout power | dynamic, internal, switching, leakage and activity coverage |
| Low-power intent | UPF/CPF, isolation/retention strategy, power-state simulation |
| Overheads | retained-state bits, retention leakage, wake energy/latency, switch/IR cost |

Until those gates are complete, use the FPGA reports only as FPGA
post-implementation estimates and do not translate them into an ASIC node claim.
